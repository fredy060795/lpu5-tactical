a-u-*                | unknown             | blume               | Unknown Affiliation
a-p-*                | pending             | raute               | Pending Affiliation

Hinweis: JavaScript und Python verwenden verschiedene interne Namen:
  JS:     a-f → friendly, a-h → hostile, a-n → neutral, a-u → unknown
  Python: a-f → rechteck, a-h → raute, a-n → quadrat, a-u → blume
Beide Varianten sind im SYMBOLS-Objekt definiert und rendern das korrekte Icon.

 3.  ATAK IMPORT – SONDERFÄLLE (cotToMarker / cot_to_marker Refinement)
================================================================================

Bedingung                                          | Ergebnis-Typ
---------------------------------------------------|---------------------------
COT hat <meshtastic> in <detail>                   | → meshtastic_node
COT ist a-f-G-U-C + how beginnt mit "h" (human)   | → tak_unit  (SA-Kreis)
COT ist a-f-* + how beginnt mit "h" (human)        | → tak_unit  (SA-Kreis)
Sonstiger ATAK-Marker (raute/rechteck/etc.)        | → cbt_raute / cbt_rechteck / ...

Priorität: <meshtastic> Detail > Human-how > Standard-Mapping

 6.  COT-TYP ERKLÄRUNG (Aufschlüsselung)
================================================================================

COT Typ Format: a-{affiliation}-G-{function}

Affiliation:
  f = friendly (freundlich)
  h = hostile  (feindlich)
  n = neutral
  u = unknown  (unbekannt)
  p = pending  (ausstehend)

Häufige COT-Typen:
  a-f-G-U-C       = Atom, Friendly, Ground, Unit, Combat
  a-h-G-U-C       = Atom, Hostile, Ground, Unit, Combat
  a-n-G-U-C       = Atom, Neutral, Ground, Unit, Combat
  a-u-G-U-C       = Atom, Unknown, Ground, Unit, Combat
  a-p-G-U-C       = Atom, Pending, Ground, Unit, Combat
  a-f-G-E-S-U-M   = Atom, Friendly, Ground, Equipment, Sensor, Unit, Meshtastic
  b-m-p-s-m        = Bits, Mapping, Point, Spot, Map
  u-d-c-e          = User-defined, Drawing, Circle, Ellipse
  u-d-r            = User-defined, Drawing, Rectangle

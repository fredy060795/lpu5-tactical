TAK Server Integration
Connect this server to a TAK Server for advanced tactical operations (similar to ATAK/WINTAK).
WinTAK / ATAK on the same machine? Use Connection Type: TCP, Host: 127.0.0.1, Port: 8087 — no certificate required. SSL (port 8089) requires a client certificate; use TCP for local connections to avoid this.
CoT Listener (Inbound ATAK Data)
The CoT listener opens a local TCP socket (port 8088) so that ATAK clients can send CoT XML directly to this server. Received events are stored as map markers and broadcast to all connected clients in real time. The listener starts automatically on server startup.

Only TCP port 8088 is required for importing data from ATAK. UDP (port 4242) and SA Multicast (239.2.3.1:6969) are optional extensions and are not needed for basic ATAK data import.
ATAK / WinTAK — recommended setup (TCP port 8088):
  1. In ATAK/WinTAK: open Settings → Network → Server Connections → Add Connection
  2. Connection type: TCP (no SSL), Address: 127.0.0.1, Port: 8088
  3. Save and connect.
Forcing ATAK to send all data to LPU5 automatically: When ATAK connects to LPU5 via Server Connections (Streaming/TCP), it automatically sends all SA data — own position, all team member positions, and shared map items — directly to LPU5 in real time. No manual per-user sharing is required; the server connection causes ATAK to broadcast all data globally to LPU5.

For ATAK on a different device: replace 127.0.0.1 with this server's IP shown at the top of this page.

SA Multicast (239.2.3.1:6969 UDP) and UDP port 4242 are not required for data import and can remain disabled.
WinTAK / ATAK Marker Sync
Since ATAK/WinTAK does not offer global data sharing, LPU5 tracks which markers were received from connected ATAK/WinTAK clients and which were created locally. Use Check Diff to compare both sides and Push Missing to ATAK to forward any LPU5 markers that ATAK clients have not yet received.
